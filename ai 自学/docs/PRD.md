### 项目PRD（MVP）—— 英国留学生二手交易平台（曼城试点）

#### 1. 背景与目标
- **目标人群（MVP）**: 在英中国留学生；中期扩展至全体在英留学生。
- **试点范围（地理/学校）**: 曼彻斯特三校：University of Manchester（曼大）、Royal Northern College of Music（RNCM）、Manchester Metropolitan University（曼城城市大学）。
- **核心价值/差异化**: 
  - B 本地化校园场景：线下当面交易为主，校区/地铁站点对接；
  - C 价格优势与快速流转：强调高性价比与高周转效率。

#### 2. 定位与策略
- **平台定位**: 校园二手交易撮合平台，MVP 不承载在线收付款；后续迭代至托管/担保（闭环 D：A 起步，后续升级到 C）。
- **品牌与域名**: 品牌名「易收」；域名后续确定。
- **语言与货币**: 仅中文界面（B）；GBP（英镑）定价。
- **注册范围**: C 不做限制（所有用户可注册），默认首屏与筛选聚焦曼城三校内容。
- **身份与信任**: 手机号（支持 +44 英国、+86 中国号段）+ 邮箱验证（MVP）。
- **交易方式**: 一口价 + 议价（聊天砍价）。
- **交付方式**: 线下当面交易；仅提供通用安全提示（不维护具体地点清单）。

#### 3. MVP 范围
- **必备功能（用户侧）**:
  - A 上架：发布/编辑/下架；
  - B 搜索+筛选：学校/城市/类目/价格/成色；
  - C 私信聊天（支持图片）；
  - D 收藏/关注/降价提醒；
  - E 评价与举报；
  - F 多图上传+自动压缩；
  - G 站内通知（新消息、成交意向、违规提醒）。
- **上架字段（保留默认）**: 标题、描述、价格、类目、成色、位置（学校/校区）、图片；可选：原价、购买时间、发票/保修图。
- **类目（MVP）**: A 教材/参考书、B 家具/家居、C 数码/电器、D 服饰/配件、E 生活用品/厨房、F 票务/乐器/其他（运营可追加）。
- **信任与安全（MVP）**: 
  - 评分/评价体系（A），
  - 轻量纠纷仲裁流程（B），
  - 纯自建轻审（C）：关键词拦截、基础图像规则（比例/敏感特征）等，命中规则进入人工复核，
  - 举报与拉黑（运营面板联动）。

#### 4. 非功能要求
- **性能**: Core Web Vitals 友好；图片懒加载与压缩；首屏 <2.5s（Wi‑Fi 环境）。
- **隐私与合规**: 
  - GDPR：明示用于手机号/邮箱的用途与保留期限；
  - 内容合规：禁止售卖违法/危险/侵权/假冒商品；
  - 仅撮合阶段：明确“平台不参与交易与资金流”的用户协议与风险提示。
- **可观测性**: 基本日志（注册、上架、消息、举报）；异常/审核告警。

#### 5. 合规边界（英国）
- **阶段化策略**:
  - MVP：A C2C 信息撮合平台（不收款，不保管资金）；
  - 迭代：B 平台参与收款（需 KYC/GDPR/跨境合规）。
- **在线支付路线图**:
  - GBP 卡/Apple Pay：Stripe；
  - Alipay：Stripe 原生渠道；
  - WeChat Pay：Adyen/Airwallex 等渠道；
  - Escrow 托管：Stripe Connect Custom/Express 或 Adyen 基于托管逻辑。

#### 6. 关键流程（MVP）
- **新用户**: 手机号验证码注册（+44/+86）→ 邮箱绑定 → 选择学校（可跳过）→ 设置昵称与头像。
- **上架**: 选择类目 → 填写标题/描述/价格/成色/位置 → 上传图片（多图，自动压缩）→ 提交 → 通过自动审核后上线（命中高风控进入人工复核）。
- **浏览与搜索**: 首页聚焦曼城三校热门 → 搜索/筛选 → 详情页 → 收藏或发起聊天。
- **聊天/议价**: 私信对话（文本/图片）→ 一键报出“接受价/还价” → 达成线下约见 → 交易完成后互评。
- **评价/举报**: 成交后互评；任何时点可举报（进入运营面板处理）。

#### 7. 禁止与限制清单（摘）
- 处方药/烟草/酒精（含未成年相关）、刀具危险品、假冒伪劣/盗版、侵权内容、违法电器改装、动物活体等；
- 高风险类目（数码、票务）加强审核与提示。

#### 8. 指标（前8周）
- 供给与成交：有效上架数、售出率、从上架到售出中位天数；
- 用户活跃：WAU/DAU、聊天发起率、收藏率；
- 体验与安全：被举报率、审核拒绝率、纠纷率；
- 增长：三校覆盖率（注册/活跃渗透）。

#### 9. 第三方与技术栈（确定）
- Web：Next.js（SSR/ISR）+ TypeScript；
- 后端：Next API Routes 或 NestJS（根据团队偏好）；
- 数据库：Postgres + Prisma（本地可用 SQLite，部署切换 Postgres）；
- 存储：Cloudflare R2（S3 兼容）+ Sharp 压缩；
- 即时聊天：Pusher；
- 短信：腾讯云 SMS（支持 +44/+86）；邮件：Postmark/Sendgrid；
- 审核：自研轻量审核（关键词+基础图像规则）；
- 部署：Vercel（Web）+ Render/Fly（后端）+ Railway/Neon（DB）。

#### 10. 主要风险与对策
- 冷启动供需不足 → 校园大使/社团合作 + 先搭建“教材/家具”两个高频供给；
- 内容与诈骗风险 → 前后端拦截、评分与仲裁、显著风险提示；
- 线下面交安全 → 明确通用安全提示（天亮/公共场所/结伴/不提前转账等）；
- 在线支付合规复杂 → 分阶段推进：先撮合，后支付，再托管。
